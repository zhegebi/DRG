# 软件工程大作业

**时间**：2026年03月

## 总体目标

利用大模型或智能体框架，构造一个医保 DRG 入组智能体。并自动构造相应的需求分析、架构设计和测试文档，将文档自动提交到虚拟文档系统。

（以下四个模块可以设计成智能体）
- 测试用例生成智能体

## 具体需求

### DRG 入组

**功能**：  
根据电子病历和入组规则自动匹配输出 DRG 分组结果。

**输入**：  
- 电子病历文本（含主要诊断、手术操作、次要诊断等）  
- DRG 分组规则（含 MDC 分类、ADRG 分组、MCC/CC 列表）

**输出**：  
- DRG 组号、组名（如 "GD15" "阑尾切除术，无合并症"）  
- 入组原因说明等

### 文档自动生成

**功能**：  
基于系统需求、代码和设计信息自动生成符合规范的文档。

**输入**：  
- 系统需求、系统代码等

**输出**：  
- 需求分析文档 - 含系统功能需求、用例等  
- 架构设计文档 - 含整体结构、模块关系等  
- 测试文档 - 含测试策略、测试方案

### 测试用例生成

**功能**：  
根据 DRG 规则自动构建测试场景并生成测试用例。

**输入**：  
- DRG 分组规则  
- 病历样本

**输出**：  
- 正常场景测试用例（不同诊断 + 手术组合）  
- 边界测试用例（合并症有无等）  
- 异常测试用例（编码错误、信息缺失等）

### 虚拟文档系统与提交

**功能**：  
- 构建一个虚拟文档系统  
- 自动接收其他智能体生成的文档  
- 文件存储与提交

## DRG 入组

### DRG 定义

按疾病诊断相关分组（Diagnosis Related Groups）。具体含义：将病人按“诊断 + 治疗方式 + 个体特征”分到不同组，每组对应一个打包付费标准。

### DRG 入组规则文件

《按病组（DRG）付费分组方案（2.0版）》

### DRG 分组结构（三层）

1. **MDC**（主要诊断大类）：根据主要诊断判断  
2. **ADRG**（核心分组）：根据主要手术 + 主要诊断判断  
3. **DRG**（细分组）：考虑合并症/严重合并症（CC/MCC）

### DRG 入组实例

**患者病历**：  
- 主要诊断：A01.002+G01*（伤寒性脑膜炎）  
- 次要诊断：J96.0（急性呼吸衰竭）  
- 主要手术：38.1000x002（动脉内膜剥脱术）

**入组过程**：  
- 主诊断为 A01.002+G01*（伤寒性脑膜炎） → 进入神经系统疾病及功能障碍大类（MDCB）  
- 主手术 38.1000x002（动脉内膜剥脱术） → 命中神经系统复合手术组（ADRG：BB1）  
- 次要诊断 J96.0（急性呼吸衰竭）：  
  - 该诊断属于 MCC（严重合并症或并发症）列表  
  - 经排除表校验，该并发症不被主诊断所排除  
  - 该 ADRG 支持并发症分层，因此 MCC 判定成立  
- 进入 **BB11** 神经系统复合手术，伴严重合并症或并发症

**分组结果**：  
- MDC：MDCB  
- ADRG：BB1  
- DRG：BB11
